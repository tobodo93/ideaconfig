Spelling Checker Plugin

Spelling Checker Plugin for IntelliJ IDEA based around the Open Source Jazzy Spell Check API.

Making the plugin and the sources available here as I've been having problems with the official IntelliJ plugins Wiki site which hosts the SpellCheck Plugin Homepage. I Can't tell wether IntelliJ intend to retire this site in favour of the new IntelliJ IDEA 4.0 Plugin Repository which exposes a basic web interface to the builtin plugin manager.

Installation

Easiest route for anybody using IDEA 4.0 is to use the builtin plugin manager (file -> settings -> plugins) and install or update the plugin from there. Othwerwise follow these basic instructions to download install/compile the plugin yourself from one of the following distributions:

IntelliJ IDEA 4.5 version

SpellCheckPlugin.zip (binary)

SpellCheckPlugin-src.zip

IntelliJ IDEA 4.0.x version

SpellCheckPlugin07.zip (binary)

SpellCheckPlugin07-src.zip

IntelliJ IDEA 3.x version only

SpellCheckPlugin04b.zip (untested backport)

  1. Unpack the SpellCheckPlugin.zip directly into your IDEA Plugins directory, so that the lib jars and dictionaries are unpacked into a SpellCheckPlugin subdirectory under your %IDEA_HOME%/plugins.
  2. Restart IntelliJ Idea to active the plugin

Usage

The plugin configuration options are available from an icon installed into the IDE Settings panel. This allows you to select some of Jazzys syntax checking options and the location (and type) of the dictionary files. Basically the disc based dictionaries are generally slower during use, whereas the memory based ones are quicker - but use a significantly amount of memory to load the dictionary file each time its needed. You also get the option to cache the memory dictionary between uses for the quickest possible operation.

To active the spell checker you've basically got two options "Spell Check Selection" and "Smart Check".

Spell Check Selection is available from the Code Menu or from the default keyboard shortcut (Ctrl + Shift + K). This checks the active selection or just the current word when no selection is available.

Alternatively Smart Check (Ctrl + Shift + D) uses the power of IntelliJ to check only the comments, javadoc and string literals within the currently active document. I.E. Checks the possibly embarrassing (when misspelt) user-facing content. Currently this is limited to Java files only (not xml, jsp etc..).

NB: Some of the options exposed by the Jazzy engine currently don't work - things like multiple word warnings etc.. haven't been implemented (or just buggy) hopefully work will continue on improving the engine. Also note that Jazzy can be downloaded and recompiled directly from their CVS, and with luck replacing the jazzy-core.jar in the plugins lib folder with the new copy should enable users to keep to-to-date themselves.
